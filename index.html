Implicit vs Explicit Tracks

								
	.container {
		display: grid;
		grid-template-columns: 200px 200px 200px 200px;
		grid-gap: 20px;
	}
  • Understanding how Implicit and Explicit grid/tracks work is a part of the fundamental understanding of CSS Grid.
  • If you do not explicitly define a tracks, they are implicit - meaning that the browser will automatically fill in the remaining items that do not fit in to the defined tracks.
  • In this example, we have explicitly defined our columns but we have not defined the rows, so they will be implicit.

Implicit vs Explicit Tracks

  • Here you can see our 4 explicitly defined columns.
  • If we did not explicitly define our columns, we would have 1 column and the width would expand to 100% of the container, as you saw in our initial example.
  • Once the defined columns are filled, the browser needs to put the remaining items somewhere so it will wrap them around, respecting the column definition, and fill them in as rows.
  • Even though we did not define any rows, they will be implicitly created.
  • Similarly, if you do define a set of grid rows, the grid extra items will fill themselves in automatically after they take up the space in the rows that have been explicitly defined.

New Units and Notations

  • Along with CSS Grid Layout, comes a couple of really helpful new units and notations:
  • The FR Unit and the repeat() function.

The FR Unit

							
	.container {
		display: grid;
		grid-template-columns: 1fr 1fr 1fr 1fr;
		grid-gap: 20px;
	}
  • Fr is a new unit of length, which is short for “fraction” or "fractal unit".
  • MDN defines it as a unit which represents a fraction of the available space in the grid container.
  • In this example, we've re-written the `grid-template-colums` to use the fr unit instead of absolute pixel units (which aren't very flexible in this case).

The repeat() Function

							
	.container {
		display: grid;
		grid-template-columns: repeat(4, 1fr);
		grid-gap: 20px;
	}
  • We can improve on this further with the repeat() notation.
  • Instead of repeating the same value over and over, we are now wrapping that value in repeat().
  • The second value of repeat() is the value that you want repeated.
  • The first value of repeat() is the number of times you want to repeat that value.
  • This allows a recurring pattern to be written in a more compact form, as you see here.

Result

Before: Pixel Units
  • Here's the result from our earlier example where the grid items are not stretching the full-width of the container.
After: Fr and repeat()
  • After we replace the pixel values with a more flexible `fr` unit, we see that the grid now takes up the full width of the container.
  • What's happeining here is the `fr` units are telling the grid to fill in all of the available space with 4 evenly spaced columns.

Mixing Units

							
	.container {
		display: grid;
		grid-template-columns: repeat(2, 150px) 1fr;
		grid-gap: 20px;
	}
  • You don't have to stick with one measurement unit for your track definitions.
  • You can, for example, mix pixels with the fr unit, like you see here. This can be handy if you want to have part of your grid take up a set amount of space and the rest of the grid to flex around it.

Result

  • The way this is calculated is the pixel values (the set/absolute space) are calculated first and then the items using `fr` are divided up with the remaining space. Another way to look at it is to think of `fr` as "Free Space", where the "Paid Space" is taken up first.

  • We've set the space between our rows and columns to be 20px, with the `grid-gap` rule.
  • We'll cover more on these terms a bit later - for now, just note that we're setting these definitions all on the parent container - we've not touched the child elements at all.

Beyond The Basics

Placing Grid Items

  • Now that we've covered the basics and a bunch of terminology, let's start placing some of our items on the grid.
  • By default, each item in a grid will be placed automatically in the default order.
  • If we want to customize our grid beyond that default layout, we can position our items anywhere on the grid using line numbers.

Standard Approach

							
	.item1 {
		grid-column-start: 1;
		grid-column-end: 3;
		grid-row-start: 1;
		grid-row-end: 3;
	}
	.item8 {
		grid-column-start: 3;
		grid-column-end: 5;
	}
	.item10 {
		grid-column-start: 1;
		grid-column-end: 2;
		grid-row-start: 3;
		grid-row-end: 4;
	}
  • Using the same container markup and styles from our previous examples (just adding a few additional items), we can start to target individual grid items for more fine-grained control over a more complex grid layout.
  • Here, we're setting specific start and end positions for grid items #1 and #8 to span them across multiple rows and columns.
  • Additionally, we're moving the position of item #10 to item #6.
  • The items we don't position will attempt to fill in and span their default height and width.

Shorthand Poperty

							
	.item1 {
		grid-column: 1 / 3;
		grid-row: 1 / 3;
	}
	.item8 {
		grid-column: 3 / 5;
	}
	.item10 {
		grid-column: 1 / 2;
		grid-row: 3 / 4;
	}
  • Here's the shorthand property where the first number in each rule represents the starting point and the last number represents the end point.

Result

  • Here's the resulting output.
  • You can see item #1 spans across 2 rows and 2 colums, item #7 spans two rows, and item #10 is in position #6.

Grid Template Areas

  • We're not limited to positioning items via line numbers - we can also use named grid areas with the `grid-template-areas` and `grid-area` properties.

Beyond The Basics, Part II

CSS

							
	.container {
		display: grid;
		grid-gap: 1.25rem;
		grid-template-columns: 200px 1fr;
		grid-template-rows: 150px 1fr 100px;
		grid-template-areas:
			"header header"
			"sidebar content"
			"footer footer"
		height: 75vh;
	}
  • This block of CSS is pretty similar to the code in our previous examples, with the main exception of the `grid-template-areas`.
  • Instead of placing each grid item individually, we can define the entire layout with this property and then assign the areas to each item using the `grid-area` property.

Beyond The Basics, Part II

CSS (continued)

							
	header {
		grid-area: header;
	}
	aside {
		grid-area: sidebar;
	}
	.content {
		grid-area: content;
	}
	footer {
		grid-area: footer;
	}

Result

  • And just like that, we have a basic page layout.
  • You may have noticed I mixed pixel and fr units to keep some of the elements locked in at a set height or width.
  • The sidebar, for example, will stay at a fixed 200px width, while the content area will expand and take up the rest of the remaining space in the container.

Beyond The Basics, Part II

CSS

							
	.container {
		display: grid;
		grid-gap: 1.25rem;
		grid-template-columns:
			[container-start sidebar-start] 200px
			[sidebar-end content-start] 1fr
			[content-end container-end];
		grid-template-rows:
			[header-start] 150px
			[main-start] 1fr
			[footer-start] 100px
			[footer-end];
		height: 75vh;
	}
  • With the same markup we used in the previous example, we can name our lines when defining our grid - allowing us to use those names instead of grid lines.
  • To name a grid line, we need to provide the name in square brackets.

  • To name a grid line, we need to provide the name in square brackets.

Dev Tools

Firefox Developer Edition

  • The last item to cover is dev tools for CSS grid.
  • CSS Grid is hard to visualize because they're made up of columns, rows, traps, and gaps - which aren't actually elements in and of themselves
  • Dev tools are helpful to visualize what is going on.
  • While Chrome's dev tools are getting better here, the best dev tools for CSS Grid right now are in Firefox Developer Edition.
  • Firefox has a Layout panel that lists all the available CSS Grid containers on the page and includes an overlay to help you visualize the grid itself.
  • You can choose what you want displayed on the overlay, including grid line numbers, area names (which we'll talk about shortly), even the color of the lines.

Grid Line Meanings

  • The solid line is the start and end of the Explicit Grid
  • The dashed diagonal lines are the Grid Gap
  • The dark, dashed lines represent an Explicit Track
  • Lastly, the lighter dotted lines represent an Implicit Track
