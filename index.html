Beyond The Basics

Placing Grid Items

  • Now that we've covered the basics and a bunch of terminology, let's start placing some of our items on the grid.
  • By default, each item in a grid will be placed automatically in the default order.
  • If we want to customize our grid beyond that default layout, we can position our items anywhere on the grid using line numbers.

Standard Approach

							
.item1 {
	grid-column-start: 1;
	grid-column-end: 3;
	grid-row-start: 1;
	grid-row-end: 3;
}
.item10 {
	grid-column-start: 1;
	grid-column-end: 2;
	grid-row-start: 3;
	grid-row-end: 4;
}

Shorthand Poperty

							
.item1 {
	grid-column: 1 / 3;
	grid-row: 1 / 3;
}
.item10 {
	grid-column: 1 / 2;
	grid-row: 3 / 4;
}
  • Using the same container markup and styles from our previous examples (just adding a few additional items), we can start to target individual grid items for more fine-grained control over a more complex grid layout.
  • Here, we're setting specific start and end positions for grid item #1 to span them across multiple rows and columns.
  • Additionally, we're moving the position of item #10 to item #6.
  • The items we don't position will attempt to fill in and span their default height and width.
  • Here's the shorthand property where the first number in each rule represents the starting point and the last number represents the end point.

Result

  • Here's the resulting output.
  • You can see item #1 spans across 2 rows and 2 colums, item #7 spans two rows, and item #10 is in position #6.

Beyond The Basics, Part II

Grid Template Areas

  • We're not limited to positioning items via line numbers - we can also use named grid areas with the `grid-template-areas` and `grid-area` properties.

The Plan

  • And just like that, we have a basic page layout.
  • You may have noticed I mixed pixel and fr units to keep some of the elements locked in at a set height or width.
  • The sidebar, for example, will stay at a fixed 200px width, while the content area will expand and take up the rest of the remaining space in the container.

  • Here's the markup, using a basic page layout with a header, sidebar, content section, and footer.

CSS

							
.container {
	display: grid;
	grid-gap: 1.25rem;
	grid-template-columns: 200px 1fr;
	grid-template-rows: 150px 1fr 100px;
	grid-template-areas:
		"header header"
		"sidebar content"
		"footer footer"
	height: 75vh;
}
							
						
							
header {
	grid-area: header;
}
aside {
	grid-area: sidebar;
}
.content {
	grid-area: content;
}
footer {
	grid-area: footer;
}
  • This block of CSS is pretty similar to the code in our previous examples, with the main exception of the `grid-template-areas`.
  • Instead of placing each grid item individually, we can define the entire layout with this property and then assign the areas to each item using the `grid-area` property.

Result

  • And just like that, we have a basic page layout.
  • You may have noticed I mixed pixel and fr units to keep some of the elements locked in at a set height or width.
  • The sidebar, for example, will stay at a fixed 200px width, while the content area will expand and take up the rest of the remaining space in the container.

Beyond The Basics, Part III

Naming Lines

  • Let's quickly go over a handful of the properties I'll be using in the upcoming examples.
  • I won't be covering ALL of them, for the sake of time, but I think it'll be helpful to touch on some of these key properties.

CSS

							
.container {
	display: grid;
	grid-gap: 1.25rem;
	grid-template-columns:
		[container-start sidebar-start] 200px
		[sidebar-end content-start] 1fr
		[content-end container-end];
	grid-template-rows:
		[header-start] 150px
		[main-start] 1fr
		[footer-start] 100px
		[footer-end];
	height: 75vh;
}
  • With the same markup we used in the previous example, we can name our lines when defining our grid - allowing us to use those names instead of grid lines.
  • To name a grid line, we need to provide the name in square brackets.

CSS (continued)

							
header {
	grid-column: container-start / container-end;
	grid-row: header-start / main-start;
}
aside {
	grid-column: sidebar-start / sidebar-end;
	grid-row: main-start / footer-start;
}
.content {
	grid-column: content-start / content-end;
	grid-row: main-start / footer-start;
}
footer {
	grid-column: container-start / container-end;
	grid-row: footer-start / footer-end;
}
  • Now, we can use our pre-defined line names when placing items, instead of the line numbers.
  • To name a grid line, we need to provide the name in square brackets.

Result

  • The result is the same as the Grid Area example.
  • Here we're spanning our items across named lines, as opposed to directly placing items in pre-defined areas.

Dev Tools

Firefox Developer Edition

  • The last item to cover is dev tools for CSS grid.
  • CSS Grid is hard to visualize because they're made up of columns, rows, traps, and gaps - which aren't actually elements in and of themselves
  • Dev tools are helpful to visualize what is going on.
  • While Chrome's dev tools are getting better here, the best dev tools for CSS Grid right now are in Firefox Developer Edition.
  • Firefox has a Layout panel that lists all the available CSS Grid containers on the page and includes an overlay to help you visualize the grid itself.
  • You can choose what you want displayed on the overlay, including grid line numbers, area names (which we'll talk about shortly), even the color of the lines.

Grid Line Meanings

  • The solid line is the start and end of the Explicit Grid
  • The dashed diagonal lines are the Grid Gap
  • The dark, dashed lines represent an Explicit Track
  • Lastly, the lighter dotted lines represent an Implicit Track
