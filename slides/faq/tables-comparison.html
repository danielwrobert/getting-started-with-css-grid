Why tables are bad for layout

  • HTML tables were designed for tabular data
  • Using tables for layout is not accessible
  • Complex markup structures result in more difficult maintainability and debugging
  • Tables are not responsive by default
  • **Down arrow to show bullet point fragments!
  • There are a few main reasons why you shouldn't use HTML tables for layouts and why CSS Grid is different:
    • HTML tables were designed for tabular data. Early on, people used them as a layout hack because CSS support across browsers was very poor.
    • Using tables for layout reduces accessibility for visually impaired users - the output from screen-readers tends to be confusing for users because of the more complex markup.
    • Because the markup structures tend to be more complex than proper layout techniques, this can result in code that is harder to maintain and debug.
    • Lastly, tables are not automatically responsive: Proper layout containers default to 100% of the width of their parent element.
    • Tables on the other hand are sized according to their content by default.
    • Because of this, extra measures are needed to get a table-based layout to work across devices.
