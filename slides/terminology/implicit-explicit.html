Implicit vs Explicit Tracks

		
/* CSS */
.container {
display: grid;
grid-template-columns: 200px 200px 200px 200px;
grid-gap: 20px;
}
  • Understanding how Implicit and Explicit grid/tracks work is a part of the fundamental understanding of CSS Grid.
  • If you do not explicitly define a track, it will be implicit - meaning that the browser will automatically fill in the remaining items that do not fit in to the defined tracks.
  • Here, we've explicitly defined 4 columns.
  • If we did not define them, we would have 1 column and the width would expand to 100% of the container<./li>
  • Once the defined columns are filled, the browser needs to put the remaining items somewhere so it will wrap them around, respecting the column definition, and fill them in as rows.
  • Even though we did not define any rows, they will be implicitly created.
  • Similarly, if you do define a set of grid rows, the grid extra items will fill themselves in automatically after they take up the space in the rows that have been explicitly defined.
