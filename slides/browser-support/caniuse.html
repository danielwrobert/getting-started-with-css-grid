  • As with any new feature, I would highly recommend you study your analytics and determine if this will work for your user-base.
  • That said, browser support for CSS Grid Layout is pretty good!
  • In addition, there are a couple of useful things to be aware of when considering fallbacks, if you need them:
    • Browsers ignore CSS they don’t understand
    • So, if a browser that doesn’t support CSS Grid comes across the a Grid-specific property, it ignores it and continues on.
    • And with that, the new layout specification knows about old layout
    • This means that if an item is floated or uses the clear property, then becomes a grid item, it will no longer display any floating or clearing behavior.
    • Between these two things, it can be failry simple to implement a fallback for an element that uses CSS Grid - you just need to determine if it's worth the overhead, based on the demographic data of your application.
